What is the display property?

The display property determines how your content in your HTML file is displayed. Most elements default as inline or block types, but you may specify for an element to be displayed as an inline-block type element instead. All of these types are interchangeable, so use them as you see fit!

Inline

CSS code display: inline;

inline elements appear next to each other in the order they are written in the HTML. These however, will not allow you adjust the top and bottom margin nor the corresponding padding attributes.

For example, anchor and span are displayed inline by default.

This link is displayed inline (<- this is a span, by the way) with the sentence. I can also increase the left and right margin and padding! It’ll ignore top and bottom margins and padding (though the highlight is still displayed).

Block

CSS code display: block;

block elements will always start on a new line. These will look at any and all margin and padding values and apply them accordingly.

For example, paragraph, list and div start on new lines.

I’m a box! I've got a 30px margin and a 30px padding all around. A block element, like me, without a defined width will stretch to the edges of the width of the container I'm in (minus the margins of course).

Inline-block

CSS code display: inline-block;

Does exactly as it says. This will set an element to be displayed inline with all the attributes of a block element available, most notably with the top and bottom margin and padding in effect.

Why is this useful?

I found this to be extremely useful for making my navigation bar. It consists of an unordered list of anchor elements.

By default, the elements will be displayed like this (this bar is just a demo with dead links):

  • Blog
  • About

<ul style="list-style: none; padding: 0;">
    <li>
        <a href="#example-nav-target" class="example-nav-list-link">
            Home
        </a>
    </li>
    <li>
        <a href="#example-nav-target" class="example-nav-list-link">
            Blog
        </a>
    </li>
    <li>
        <a href="#example-nav-target" class="example-nav-list-link">
            About
        </a>
    </li>
</ul>

Declaring the list to display with and inline-block will give us this:

  • Blog
  • About

<ul style="list-style: none; padding: 0;">
    <li style="display: inline-block;">
        <a href="#example-nav-target" class="example-nav-list-link">
            Home
        </a>
    </li>
    <li style="display: inline-block;">
        <a href="#example-nav-target" class="example-nav-list-link">
            Blog
        </a>
    </li>
    <li style="display: inline-block;">
        <a href="#example-nav-target" class="example-nav-list-link">
            About<
        /a>
    </li>
</ul>

That’s pretty much it on how I made my navigation bar, but there was just one little thing I didn’t like about this. Did you notice the small, unclickable spaces in between the links? They exist because inline-block lines the elements up like a sentence, so they are essentially the same spaces that exist when we type.

There are a few workarounds for this issue. For me, I removed the closing tags </li> in my HTML and this fixed the gaps:

  • Blog
  • About 

<ul style="list-style: none; padding: 0;">
    <li style="display: inline-block;">
        <a href="#example-nav-target" class="example-nav-list-link">
            Home
        </a>
    <li style="display: inline-block;">
        <a href="#example-nav-target" class="example-nav-list-link">
            Blog
        </a>
    <li style="display: inline-block;">
        <a href="#example-nav-target" class="example-nav-list-link">
            About<
        /a>
</ul>
